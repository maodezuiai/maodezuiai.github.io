Unity Shader

发表于 2020-08-31 更新于 2020-09-08 分类于 图形学

概述

材质


材质位置

Shader

  1. Standard surface Shader：包含标准准光照模型的表面着色器模板（基于物理的）
  2. Unlit Shader：产生一个不包含光照（但包含雾气的）的基本的顶点/片源着色器
  3. Image Effect Shader：提供了后期屏幕后期处理效果的模板
  4. Compute Shader：利用GPU的并行性进行与常规渲染流水线无关的

Unity Shader的结构

Shader的名称和位置

Shader “Custom/MyShader”{ }

连接材质和Shader

Properties语义块

  • Name 属性名
  • display name 材质面板名称
  • PropertyType 属性类型（见表）
  • DefaultValue 默认属性值
  • Int Float Range 值是单独数字
  • Color Vector （0，0，0，0）
  • 2D Cube 3D “纹理名称”{} ^[纹理名称可为空]

    Properties支持重载，其目的仅仅是让属性出现在材质面板中

SubShader语块（重要）

  • Tags和RenderSetup也可以在pass中声明，但是只有Tags的设置不同
  • 如果在Subshader中声明，会影响后面全部的Passes
  • Pass越多性能越差

RenderSetup

如果不行影响所有pass（如分别渲染正反面），可在有需求的pass中单独声明

Tags

Tags 是一个键值对

Tags{“Tagname” = “Valuel” “TagName2” = “Value2”}

图中的只能用于SubShader中

Pass语义块

  1. 在Pass中定义Pass名称

    Name ”MyPassName”（必须为全部大写）

  2. 通过ShaderLab的UsePass使用其它UnityShader的Pass

    UsePass ”MyShader/MYPASSNAME”

通过上述方式可以复用其它Passes，降低能耗

Pass中的标签类型

除这些外还有：

  • UsePass:复用其它Shader的Pass
  • GrabPass：抓取屏幕存储在一张纹理中

Fallback

Fallback”name”
//或者
Fallback off

用来指认最低级的shader是哪个

其它的语义（很少用）

  • CustonEditor 扩展编辑界面
  • Category 编组

Unity Shader的形式

UnityShader的主要任务是指定各种着色器所需的代码，可以写在SubShader中（表面着色器），也可以写在Pass中（顶点/片元着色器和固定函数）

Surface Shader（表面着色器）

Unity自己创造的着色器，代码量很少但渲染代价较大，就是帮助处理了很多的光照信息

从上面例子看出：

  • 表面着色器定义在SubShader（非Pass）中的CGPROGRAM和ENDCG中
  • 不需要关心Pass有多少，如何渲染等问题，Unity会自动处理
  • 用CG/HLSL编写
  • 背后仍被编译成对应的Vertex/Fragment Shader

Vertex/Fragment Shader（顶点/片元着色器）

简单的例子：

  • 写在CGPROGRAM和ENDCG中
  • 需要设置每一个Pass，代码量上升但是也更灵活
  • 用CG/HLSL编写

Fixed Function Shader（固定函数着色器）

简单的例子:

  • 写在Pass中，代码相当于Pass的渲染设置
  • 绝大多数GPU支持可编程渲染管线，逐渐淘汰
  • 使用ShaderLab语法编写，而不是CG/HLSL
  • 在5.2版本后，在后台处理时都会编译成对应的Vertex/Fragment Shader
