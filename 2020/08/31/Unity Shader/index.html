Unity Shader

发表于 2020-08-31 更新于 2020-09-07 分类于 图形学

概述

材质

Shader

  1. Standard surface Shader：包含标准准光照模型的表面着色器模板（基于物理的）
  2. Unlit Shader：产生一个不包含光照（但包含雾气的）的基本的顶点/片源着色器
  3. Image Effect Shader：提供了后期屏幕后期处理效果的模板
  4. Compute Shader：利用GPU的并行性进行与常规渲染流水线无关的

Unity Shader的结构

Shader的名称和位置

Shader “Custom/MyShader”{ }

连接材质和Shader

Properties语义块

properties {
Name(“display name”,PropertyType) = DefaultValue
Name(“display name”,PropertyType) = DefaultValue
//更多属性
}

  • Name 属性名
  • display name 材质面板名称
  • PropertyType 属性类型（见表）
  • DefaultValue 默认属性值
  • Int Float Range 值是单独数字
  • Color Vector （0，0，0，0）
  • 2D Cube 3D “纹理名称”{} ^[纹理名称可为空]

    Properties支持重载，其目的仅仅是让属性出现在材质面板中

SubShader语块（重要）

SubShader {
    //可选的
    [Tags]

    //可选的
    [RenderSetup]

    Pass{
    }
    // Other Passes
}

  • Tags和RenderSetup也可以在pass中声明，但是只有Tags的设置不同
  • 如果在Subshader中声明，会影响后面全部的Passes
  • Pass越多性能越差

RenderSetup

如果不行影响所有pass（如分别渲染正反面），可在有需求的pass中单独声明

Tags

Tags 是一个键值对

Tags{“Tagname” = “Valuel” “TagName2” = “Value2”}

图中的只能用于SubShader中

Pass语义块

Pass{
    [Name]
    [Tags]
    [RenderSetup]
    //Other code
}

  1. 在Pass中定义Pass名称

    Name ”MyPassName”（必须为全部大写）

  2. 通过ShaderLab的UsePass使用其它UnityShader的Pass

    UsePass ”MyShader/MYPASSNAME”

通过上述方式可以复用其它Passes，降低能耗

Pass中的标签类型

除这些外还有：

  • UsePass:复用其它Shader的Pass
  • GrabPass：抓取屏幕存储在一张纹理中
