Unity Shader基础

概述

UnityShader！= Shader

  • 传统Shader中可以编写特定的类型的Shader如顶点/片元着色器（分开编写）
    Unity Shader可以包含在同一个文件中
  • 传统Shader无法设置渲染设置，需要其它代码设置
    Unity Shader则可以
  • 传统Shader需要小心处理输入输出关心
    Unity Shader则只需声明属性即可，并且提供了模型自身属性的直接访问方式，不需要另外编码传递给着色器
  • 传统Shader更具有灵活性
    Unity Shader因为高度封装，对有些着色器的支持就差点

    Unity Shader和CG/HLSL的关系

    Unity Shader是用ShaderLab编写的，而ShaderLab可以内嵌CG/HLSL（下方案例中会指出使用方法）

    需要注意的一点时，CGHLSL是写在Pass内的，Surface Shader其实是一种抽象，后台依然会转为Vertex/Fragment Shader的形式

材质


材质位置

Shader

  1. Standard surface Shader：包含标准准光照模型的表面着色器模板（基于物理的）
  2. Unlit Shader：产生一个不包含光照（但包含雾气的）的基本的顶点/片源着色器
  3. Image Effect Shader：提供了后期屏幕后期处理效果的模板
  4. Compute Shader：利用GPU的并行性进行与常规渲染流水线无关的

  • 在5.2版本后，在后台处理时都会编译成对应的Vertex/Fragment Shader

Unity提供的Cg/HLSL语义

  • 语义实际就是一个赋给Shader输入和输出的字符串，这些语义可以让Shader知道从哪里读取数据，又将数据输出在哪里。
  • 通常情况下输入和输出由我们自己决定（COLOR0描述color，color本身有什么数据并不重要）。
  • 有些语义有特殊含义，并且在不同的位置含义不同（TEXCOORD0在a2v中是将第一套纹理传入texcoord中，而在v2f中则无特别含义，可自己定义）。
  • DirectX 10，增加了系统数值语义，以系统数值SV（system-value）开头，这些语义都有着特殊含义，描述的变量不能随意赋值。
  • 为了更好的跨平台，尽量使用SV的语义（如PS4）。

Unity中的基础光照

如何看到光

光源

光学中用辐照度来量化光。对于平行光，辐照度时通过计算出至于光源l的单位面积上单位时间里穿过的能量来得到。
Shader中可以利用l和n的余弦值来得到。


辐照度和照射到物体表面时光线的距离d/cosθ成反比，所以和cosθ成正比，而cosθ又可以通过l和n的点积得到。

吸收和散射

  • 散射只改变光的方向，不改变光的强弱。
  • 吸收只改变光的强弱，不改变光的方向。
    光线在物体表面散射后有两种方向：
  • 折射（refraction）或透射（transmission）：光被散射到内部。
  • 另一种是反射（reflection）：光被散射到外部。
  • 对于不透明物体，有些光会先折射到物体内，最后再发射出物体，而另一些被吸收。
    为了区分这两种反射方式，在光照模型中使用了不同的部分来计算：
  • 高光反射(specular)：反射光线。
  • 漫反射(diffuse)：表示有多少光被折射、吸收、反射出表面。

根据入射光线的数量和方向可以算出出射光线的数量和方向，用出射度(exitance)来描述。
辐照度和出射度是线性关系，而他们的比值就是材质的漫反射属性和高光反射属性。不理解

BRDF光照模型

当给定物体表面一个点时，BRDF包含了对这个点的外观的完整描述。大多数用一个数学公式表示，并提供一些参数修改材质属性。通俗讲就是可以给出在某个出射方向上的光照能量分布。本章都是简化的标准模型。
谨记：图形学第一定律，如果它看起来是对的，那么它就是对的。

标准光照模型

基本理念：直接光照，就是光照射物体后直接反射一次到摄像机中。
把进入摄像机的光线分为四类：

  • 自发光（emissive）:$c_{emissive}$ 在没有全局光照找下自身会亮，但不会照亮周围。
  • 高光反射（specular）：$c_{specular}$ 完全反射光线。
  • 漫反射（diffuse）：$c_{diffuse}$ 向每个方向散射。
  • 环境光（ambient）：$c_{ambient}$ 用来描述间接照明。

环境光

环境光模拟间接照明：

$c_{ambient}$ = $g_{ambient}$

自发光

直接使用该材质的自发光颜色：

$c_{emissive}$ = $m_{emissive}$

漫反射

漫反射中，视角位置不重要，因为反射是随机的，可以简化认为任何反射方向上的分布都是一样的，但是入射角度很重要。
漫反射光照符合兰伯特定律（Lambert’s law）:
反射光线的强度与表面法线和光源方向之间的夹角余弦成正比。

$c_{diffuse}$ = $(c_{light} \cdot{} m_{diffuse})$ $\max(0, \hat{n}\cdot{}\hat{l})$

  • $\hat{n}$：表面法线。
  • $\hat{l}$：指向光源的矢量单位。
  • $m_{diffuse}$：材质的漫反射颜色。
  • $c_{light}$：光源颜色。
  • 要是背面不被照亮，截取到0。

高光反射

利用Phong模型计算高光反射
经验模型，并不符合现实高光反射现象。它是完全的镜面反射。


计算反射方向：

$\hat{r} = 2(\hat{n}\cdot\hat{l})\hat{n} - \hat{l}$

利用Phong模型计算高光反射

$c_{specular} = (c_{light}\cdot{}m_{specular})\max(0, \hat{v}\cdot{}\hat{r})^{m_{gloss}}$

  • $m_{gloss}$：光泽度（反光度shininess），控制高光亮点的宽度，值越大亮点越小。
  • $m_{specular}$：材质的高光反射颜色，用于控制材质对高光反射的强的和颜色。
    *$m_{light}$：光源的颜色和强度。

利用Blinn模型计算高光反射


Blinn简单的修改方法来得到类似的效果，基本思想是避免计算反射$\hat{r}$，而引入了$\hat{h}$。

$\hat{h} = \frac{\hat{v} + \hat{l}}{|\hat{v} + \hat{l}|}$

Blinn模型：

$c_{specular} = (c_{light}\cdot{}m_{specular})\max(0, \hat{v}\cdot{}\hat{h})^{m_{gloss}}$

逐像素和逐顶点

  • 逐像素光照中，以像素为基础得到法线（可以是顶点插值得到或者是纹理贴图采样得到），再进行计算，这中在面片之间对顶点法线进行插值的技术被称为Phong着色（Phong shading），不同于Phong光照模型。
  • 逐顶点光照中，在每个顶点计算光照，在渲染图元时进行线性插值，最后输出像素颜色。
  • 计算量变小，但是会出现明显的棱角现象。高洛德着色（Gouraud shading）。

总结

以上的模型都是经验模型，不能完全模拟真实情况，但是因为便于计算被广泛应用。

  • 重要的物理现象并不能模拟出来，如菲涅尔反射（Fresnel reflection）。
  • Blinn-Phong模型是各向同性（isotropic）的，当我们固定视角旋转表面时，反射不会发生变化，但有些材质需要各向异性（anisotropic），如拉丝金属和头发等。
